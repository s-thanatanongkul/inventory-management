Data Flow

1

User selects a filter

FilterBar.vue updates the global useFilters() composable — a singleton shared across all views. Reactive refs update: selectedPeriod, selectedLocation, selectedCategory, selectedStatus.

2

Component detects change

Each view watch()es the filter refs. On change, it calls the relevant api.js method (e.g., api.getOrders(filters)).

3

api.js builds the request

client/src/api.js converts the filters object to URL query params, silently skipping any value equal to 'all'. Sends an Axios GET to http://localhost:8001.

4

FastAPI filters in-memory data

apply_filters() and filter_by_month() in main.py iterate the in-memory list loaded from JSON. Supports direct months (2025-01) and quarters (Q1-2025). Never mutates the original list.

5

Pydantic validates the response

The response_model on each endpoint enforces the shape (e.g., List[Order]). FastAPI serialises to JSON and returns 200 OK.

6

Vue renders the result

The component assigns the response array to a ref(). Computed properties apply any client-side transforms (search, sort). The template re-renders reactively via v-for.

API Endpoints

Method Path Query Params Description
GET /api/inventory warehouse, category Stock levels — no time dimension
GET /api/orders warehouse, category, status, month Orders list with full filter support
GET /api/dashboard/summary warehouse, category, status, month Aggregated KPIs (revenue, orders, backlog)
GET /api/demand — Demand forecasts (next 30 days)
GET /api/backlog — Delayed orders with purchase-order flag
GET /api/spending/summary — Total procurement, operational, labor, overhead
GET /api/spending/monthly — 12-month spending breakdown
GET /api/spending/categories — Spending by category
GET /api/spending/transactions — Recent transaction list
GET /api/reports/quarterly — Q1–Q4 aggregated stats
GET /api/reports/monthly-trends — Month-over-month comparison
